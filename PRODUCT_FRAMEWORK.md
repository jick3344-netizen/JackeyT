# 老黄的品牌三问：产品框架 V2

副标题：品牌战略诊断工具

## 一、产品定义

### 1. 产品目标

帮助个人品牌和企业品牌回答三个核心问题：

1. **品牌定位：你是谁？**
2. **品牌价值：消费者为什么选择你？**
3. **品牌信任状：消费者为什么相信你？**

产品通过结构化问卷、公开资料采集和 AI 分析，形成一份有评分、有证据、有结论、有行动建议的完整品牌诊断报告。

### 2. 首期产品形态

- 以响应式独立站为主体，同时适配电脑和手机。
- 采用邀请制访问，只有获得邀请码或账号权限的用户才能使用。
- 第一阶段个人品牌与企业品牌同时上线。
- 内测期暂不收费，先验证诊断模型、报告质量和付费意愿。
- 第二阶段再开发微信小程序入口，与独立站共用账号、订单、报告和后台。

### 3. 已确认的产品决策

- 综合评分权重采用：品牌定位 35 分、品牌价值 35 分、品牌信任状 30 分。
- 个人品牌与企业品牌在首期同时上线。
- 完整报告首期由 AI 直接生成和交付，不强制人工审核。
- 系统预留人工复核状态、复核入口和报告版本记录，后期可按产品或订单开启。
- 免费预检只展示综合分、三个一级维度分数、维度状态和简短说明。
- 首期为邀请制内测，不接入正式支付。
- 首期不限定服务行业，问卷采用通用品牌框架。
- 正式名称为“老黄的品牌三问”，副标题为“品牌战略诊断工具”。

### 4. 目标用户

#### 个人品牌

- 创始人、企业家和管理者
- 专家、顾问、讲师和自由职业者
- 内容创作者和知识型 IP
- 希望建立职业影响力的专业人士

#### 企业品牌

- 初创企业和成长型企业
- 正在升级品牌的传统企业
- 新产品、新业务或新品牌项目
- 需要品牌体检的成熟企业

## 二、三维诊断模型

### 1. 品牌定位：35 分

诊断品牌能否清楚回答“我是谁，我为谁解决什么问题”。

| 二级指标 | 权重 | 核心判断 |
| --- | ---: | --- |
| 目标对象清晰度 | 8 | 是否明确服务谁，而不是面向所有人 |
| 品类与身份认知 | 7 | 用户能否快速理解品牌属于什么领域 |
| 核心问题聚焦度 | 7 | 是否聚焦一个重要、具体的问题 |
| 差异化程度 | 8 | 与同类品牌相比是否具有明确区别 |
| 表达一致性 | 5 | 各渠道是否持续传递相同认知 |

### 2. 品牌价值：35 分

诊断品牌能否清楚回答“消费者为什么要选择你”。

| 二级指标 | 权重 | 核心判断 |
| --- | ---: | --- |
| 用户需求重要度 | 7 | 解决的问题是否真实且重要 |
| 功能价值明确度 | 7 | 用户能够获得什么实际结果 |
| 独特价值强度 | 8 | 价值是否难以被替代或复制 |
| 情感与身份价值 | 5 | 是否带来认同、体验或身份表达 |
| 价值表达清晰度 | 5 | 能否用简洁语言让用户理解 |
| 价值兑现能力 | 3 | 产品和服务是否支撑价值承诺 |

### 3. 品牌信任状：30 分

诊断品牌能否清楚回答“消费者为什么相信你能兑现价值”。

| 二级指标 | 权重 | 核心判断 |
| --- | ---: | --- |
| 专业资质与背景 | 5 | 是否具备相关经验、资质或专业能力 |
| 产品与方法体系 | 6 | 是否形成稳定、可解释的交付体系 |
| 客户案例与成果 | 7 | 是否有真实案例和结果证明 |
| 第三方认可 | 4 | 是否获得媒体、机构、行业或专家背书 |
| 用户评价与口碑 | 4 | 是否有持续、具体、可信的用户反馈 |
| 长期一致性 | 4 | 言行、产品和服务是否长期一致 |

### 4. 成熟度等级

| 分数 | 等级 | 含义 |
| ---: | --- | --- |
| 85-100 | A 级：品牌引领 | 定位、价值和信任资产形成较强协同 |
| 70-84 | B 级：品牌成长 | 基础较好，但存在明显的提升环节 |
| 55-69 | C 级：品牌成形 | 已有品牌雏形，认知和证据仍较分散 |
| 40-54 | D 级：品牌模糊 | 核心表达不清，尚未形成稳定选择理由 |
| 0-39 | E 级：品牌起步 | 缺乏完整的品牌定义和信任基础 |

### 5. 评分原则

- 每个指标由多个问题共同判断，不因单个答案直接得出结论。
- 用户陈述、公开资料和实际证据分别标注，不混为一谈。
- AI 无法验证的信息标记为“待验证”，不作为强信任证据。
- 缺少资料不等于品牌一定较差，但会降低“诊断完整度”和相应指标的可信度。
- 报告同时显示“品牌得分”和“诊断可信度”，避免伪精确评分。

## 三、通用诊断流程

### 阶段一：基础信息

1. 选择个人品牌或企业品牌。
2. 填写品牌名称、行业、发展阶段和主要业务。
3. 提供官网、公众号、视频号、小红书、抖音等公开渠道。
4. 选择本次诊断最希望解决的问题。

### 阶段二：结构化问卷

- 用户完成定位、价值和信任状三组问题。
- 问卷采用选择题、量表题、短文本、多项填写和材料上传的混合形式。
- 支持暂存、继续填写和修改答案。
- 核心诊断题预计 12-18 分钟完成。
- 案例、资质、数据和附件作为条件题与证据补充，完整填写预计 20-30 分钟。

### 阶段三：资料与证据

- 用户可上传品牌介绍、宣传材料、案例、客户评价和资质文件。
- AI 提取资料中的品牌表述和信任证据。
- 首期可以先支持文本、网址和 PDF，后续再扩展更多平台采集。

### 阶段四：AI 追问

- 当答案过于抽象、矛盾或缺少证据时，AI 追加 3-8 个问题。
- 追问只围绕影响评分的关键信息，不无限延长流程。

### 阶段五：报告生成

- 输出综合得分、三维评分和诊断可信度。
- 给出判断依据、主要问题、建议表达和行动计划。
- 完整报告生成后保存到用户账户。

## 四、问卷交互规则

### 1. 题型使用原则

| 题型 | 适合的信息 | 使用原则 |
| --- | --- | --- |
| 单选题 | 阶段、主要目标、主要类型、是否具备某项能力 | 选项互斥时使用 |
| 多选题 | 渠道、价值类型、信任资产、用户选择理由 | 限制最多选择 3-5 项，并提供“其他” |
| 五级量表 | 清晰度、一致性、稳定性和成熟程度 | 用统一量表便于形成可比较数据 |
| 单行填写 | 姓名、品牌名称、职业、网址、一句话表达 | 答案较短且目标明确 |
| 多行填写 | 差异化、价值说明、经历、问题原因 | 需要用户提供语义内容和判断依据 |
| 固定多项填写 | 三个问题、三个理由、三个案例 | 显示三个独立输入框，避免内容混杂 |
| 可重复信息组 | 客户案例、资质、媒体报道 | 每组使用名称、说明、结果、链接或附件 |
| 文件或链接 | 资质、案例、客户评价和公开资料 | 作为证据，不因只勾选“有”就获得高分 |

### 2. 通用量表

一致性和成熟度问题统一采用五级量表：

1. 完全不符合 / 尚未建立
2. 较不符合 / 刚刚开始
3. 部分符合 / 偶尔做到
4. 比较符合 / 基本稳定
5. 完全符合 / 已形成体系

### 3. 问卷体验

- 每页只展示 3-5 道题，并显示整体进度。
- 选择“其他”时展开补充输入框。
- 选择“没有、不确定、尚未统计”时，不强迫用户编造答案。
- 开放题提供填写示例，但示例不进入用户答案。
- 固定多项填写题允许填写 1-3 项，不要求为了填满而虚构。
- AI 追问只针对关键信息缺口，最多追加 8 道题。
- 问卷根据前序答案动态跳题，没有某类信任资产时不展示对应的详细填写表。
- 题目分为“核心必答”“条件必答”和“证据选填”，页面上明确标识。

## 五、个人品牌诊断问卷

### A. 基础信息

| 问题 | 题型 | 选项或交互 |
| --- | --- | --- |
| 你的姓名是什么？ | 单行填写 | 必填 |
| 你的个人品牌名称是什么？ | 单行填写 | 可选；未单独命名可沿用姓名 |
| 你目前最主要的职业身份是什么？ | 单行填写 | 如品牌顾问、律师、设计师 |
| 你所在的专业领域是什么？ | 单行填写 | 支持行业搜索和手动填写 |
| 你目前主要提供什么？ | 多选题 | 咨询服务、专业服务、课程培训、内容产品、实体产品、社群会员、演讲合作、其他；最多选 3 项 |
| 你的个人品牌处于哪个阶段？ | 单选题 | 尚未开始、刚刚起步、已有内容但定位不清、已有稳定业务、正在升级或转型 |
| 用户主要通过哪些渠道认识你？ | 多选题 | 熟人推荐、线下活动、公众号、视频号、小红书、抖音、微博、知乎、B站、官网、媒体报道、其他 |
| 你最希望通过个人品牌实现什么？ | 多选题 | 提升专业影响力、获得客户、提高客单价、推广产品、建立社群、获得合作机会、职业转型、其他；最多选 3 项 |
| 本次最希望解决什么问题？ | 多选题 | 定位不清、价值表达不清、缺少差异化、内容方向分散、用户不信任、转化较低、品牌与业务不一致、其他；最多选 3 项 |

### B. 品牌定位：你是谁？

| 问题 | 题型 | 选项或交互 |
| --- | --- | --- |
| 你最希望被哪一类人认识和记住？ | 多行填写 | 提示填写人群、阶段、角色或需求特征 |
| 这些人目前最需要解决的三个问题是什么？ | 三项填写 | 分别显示“问题 1、问题 2、问题 3” |
| 其中你最擅长解决哪一个？ | 单选 + 补充 | 从上述三个问题中选择一个，并说明原因 |
| 如果只能保留一个专业身份，你希望是什么？ | 单行填写 | 核心身份题，必填 |
| 你希望别人如何用一句话介绍你？ | 单行填写 | 限 60 字 |
| 你的目标对象目前是否清晰？ | 五级量表 | 从“任何人都可以”到“有清晰具体的人群定义” |
| 与同领域的人相比，你最大的不同是什么？ | 多行填写 | 开放题，要求填写具体差异 |
| 你的差异主要来自哪里？ | 多选题 | 经历背景、专业方法、服务对象、产品设计、内容观点、资源网络、个人风格、成果表现、其他；最多选 3 项 |
| 哪些客户或工作不属于你的服务范围？ | 多行填写 | 用于判断边界是否清楚 |
| 主页简介、内容主题、产品服务和实际交付是否表达同一个身份？ | 五级量表 | 一致性自评 |
| 请提供目前使用的个人介绍。 | 多行填写 | 支持粘贴文本 |
| 请提供主要主页或公开渠道。 | 可重复链接 | 平台名称 + 网址，最多 5 个 |

### C. 品牌价值：为什么选择你？

| 问题 | 题型 | 选项或交互 |
| --- | --- | --- |
| 用户购买后最重要的变化是什么？ | 多行填写 | 要求描述购买前与购买后的差异 |
| 这个变化对用户为什么重要？ | 多行填写 | 开放题 |
| 你主要提供哪些价值？ | 多选题 | 提升效果、节省时间、降低成本、降低风险、提供便利、增强信心、获得认同、改善体验、创造机会、其他；最多选 3 项 |
| 其中最核心的价值是哪一个？ | 单选题 | 从上一题已选内容中选择 |
| 与替代方案相比，你的优势是什么？ | 三项填写 | 分别填写优势 1、优势 2、优势 3 |
| 用户还可能选择哪些替代方案？ | 多选题 | 其他专业人士、机构品牌、免费内容、内部解决、暂不处理、其他 |
| 你的专业方法中哪些部分具有独特性？ | 多行填写 | 若没有明确方法，可选择“尚未形成” |
| 你希望带给用户怎样的感受或身份认同？ | 多选 + 补充 | 安心、掌控感、专业感、成长感、归属感、品位、成就感、其他；最多选 3 项 |
| 用户最常选择你的三个理由是什么？ | 三项填写 | 理由 1、理由 2、理由 3；允许选择“不确定” |
| 用户最常犹豫或拒绝的三个原因是什么？ | 三项填写 | 原因 1、原因 2、原因 3；允许选择“不确定” |
| 你的价值承诺能否稳定兑现？ | 五级量表 | 从“主要依赖临场发挥”到“有稳定产品和交付体系” |
| 请用一句话表达你的核心价值。 | 单行填写 | 限 80 字；AI 后续帮助优化 |

### D. 品牌信任状：为什么相信你？

| 问题 | 题型 | 选项或交互 |
| --- | --- | --- |
| 你在该领域有多少年经验？ | 单选题 | 少于 1 年、1-3 年、4-6 年、7-10 年、10 年以上 |
| 你目前具备哪些信任资产？ | 多选题 | 学历背景、职业资质、从业经历、代表客户、成功案例、原创方法、作品或著作、客户评价、复购转介绍、媒体报道、奖项荣誉、机构合作、其他 |
| 请填写最重要的履历、资质或代表经历。 | 可重复信息组 | 名称 + 时间 + 说明 + 附件，最多 5 项 |
| 你是否有明确的方法、流程或服务标准？ | 单选题 | 没有、主要凭经验、有基本流程、有完整方法、有标准化且持续优化的体系 |
| 请简要说明你的方法或流程。 | 多行填写 | 选择“有”时显示 |
| 请提供三个代表性案例。 | 三组案例 | 每组包含客户类型、问题、解决方式、结果、是否可公开、证明材料 |
| 你有哪些可验证成果？ | 多选题 | 效率提升、收入增长、成本降低、风险降低、满意度、复购率、推荐率、传播数据、获奖或认证、其他 |
| 你是否系统收集客户评价和业务数据？ | 单选题 | 没有、零散保存、定期收集、系统管理并持续使用 |
| 你获得过哪些第三方认可？ | 可重复信息组 | 类型、名称、时间、链接或附件 |
| 你持续输出专业内容多久？ | 单选题 | 尚未持续输出、少于半年、半年至 1 年、1-3 年、3 年以上 |
| 你主要输出哪些内容？ | 多选题 | 观点、方法、案例、教程、行业研究、作品展示、日常记录、其他 |
| 目前最需要补强什么信任证明？ | 多选题 | 资质、方法体系、案例、结果数据、客户评价、第三方认可、内容积累、一致性、其他；最多选 3 项 |

## 六、企业品牌诊断问卷

### A. 基础信息

| 问题 | 题型 | 选项或交互 |
| --- | --- | --- |
| 企业名称和品牌名称是什么？ | 双单行填写 | 企业名称 + 对外品牌名称 |
| 所属行业是什么？ | 搜索选择 + 填写 | 支持“其他行业” |
| 主要面向什么市场？ | 多选题 | 企业客户、政府或机构、个人消费者、渠道商、平台用户、其他 |
| 主要经营区域是什么？ | 多选题 | 本地、区域、全国、海外；可补充地区 |
| 核心产品或服务是什么？ | 三项填写 | 核心业务 1、2、3 |
| 企业成立多久？ | 单选题 | 少于 1 年、1-3 年、4-6 年、7-10 年、10 年以上 |
| 企业规模如何？ | 单选题 | 1-10 人、11-50 人、51-200 人、201-1000 人、1000 人以上 |
| 品牌目前处于哪个阶段？ | 单选题 | 新建品牌、市场验证、快速成长、稳定经营、品牌升级、业务转型 |
| 主要获客和销售渠道有哪些？ | 多选题 | 销售团队、经销渠道、电商平台、线下门店、官网、内容平台、广告投放、招投标、客户推荐、其他 |
| 本次最希望解决什么问题？ | 多选题 | 定位不清、价值表达不清、同质化、认知度低、信任不足、转化低、品牌老化、多业务混乱、其他；最多选 3 项 |

### B. 品牌定位：你是谁？

| 问题 | 题型 | 选项或交互 |
| --- | --- | --- |
| 品牌最重要的目标客户是谁？ | 多行填写 | 填写客户类型、规模、角色、阶段等 |
| 目标客户会遇到哪三个关键问题？ | 三项填写 | 问题 1、2、3 |
| 品牌属于什么品类？ | 单行填写 | 用户最容易理解的品类名称 |
| 用户通常会把你和哪些品牌或方案比较？ | 三项填写 | 竞争品牌或替代方案 1、2、3 |
| 品牌最希望占据用户怎样的认知？ | 单行填写 | 限 60 字 |
| 与主要竞争者相比，三个重要差异是什么？ | 三项填写 | 差异 1、2、3，并可补充证据 |
| 差异主要来自哪里？ | 多选题 | 产品、技术、成本、效率、服务、渠道、供应链、设计、文化、资源、商业模式、其他；最多选 3 项 |

| 请提供官网和主要公开渠道。 | 可重复链接 | 平台名称 + 网址，最多 8 个 |

| --- | --- | --- |

| 与主要竞争者相比，品牌在哪些方面更优？ | 三组对比 | 对比维度 + 品牌表现 + 对手表现 + 证据 |

| 客户流失或未成交的三个主要原因是什么？ | 三项填写 | 允许选择“不确定或未统计” |

## 七、AI 诊断规则

### 1. 信息分类

AI 将收集到的信息分为四类：

- **品牌主张**：品牌自己声称的定位、价值和优势。
- **事实证据**：资质、数据、案例、产品和服务机制。
- **外部证据**：客户评价、媒体报道和第三方认可。
- **待验证信息**：暂时只有陈述，缺少资料支持的内容。

### 2. 关键判断

- 定位是否过宽、过泛或同时存在多个核心身份。
- 价值表达是否只是产品特点，而不是用户获得的结果。
- 差异化是否属于所有竞争者都可以声称的通用表述。
- 信任状是否与价值承诺直接相关。
- 案例是否只有客户名称，而缺少问题、过程和结果。
- 三个维度之间是否形成逻辑闭环。

### 3. 闭环标准

一个成立的品牌逻辑应当能够连成一句完整的话：

> 我们是面向【目标对象】的【品牌身份】，通过【独特方式】提供【核心价值】，并由【关键证据】证明我们能够持续兑现。

## 八、AI 模型与 API 接入方案

### 1. 首版模型建议

首版建议使用 OpenAI Responses API，并将模型配置做成可替换项。

| 任务 | 首版建议 | 说明 |
| --- | --- | --- |
| 问卷完整度检查、信息分类 | 较低成本模型 | 负责识别缺失项、整理品牌主张和证据 |
| 关键追问生成 | 较低成本模型 | 只针对影响评分的信息缺口生成追问 |
| 三维评分和完整报告 | `gpt-5.5` | 用于需要综合推理、判断一致性和形成报告的核心步骤 |
| 后续降本方案 | 小模型预处理 + 主模型终审 | 不牺牲核心报告质量的前提下降低成本 |

如果首期调用量不大，可以先全部使用 `gpt-5.5`，减少流程复杂度。积累真实报告和评价数据后，再把简单任务切换到较低成本模型。

模型名称不能写死在前端。服务端通过 `AI_MODEL_REPORT`、`AI_MODEL_EXTRACT` 等配置项控制，后期更换模型不影响问卷和报告页面。

### 2. API 密钥如何提供

- API 密钥只保存在独立站服务端的环境变量或部署平台的加密配置中。
- 环境变量建议命名为 `OPENAI_API_KEY`。
- 密钥不能写进网页代码、手机端代码、数据库公开字段或代码仓库。
- 浏览器只调用独立站自己的 `/api/diagnoses/*` 接口，不直接调用模型供应商。
- 正式上线时增加调用额度、单用户频率、超时、失败重试和费用告警。
- API 密钥在部署时由网站所有者录入部署平台的私密环境变量，不通过聊天、普通表单或前端管理页面传递。

数据流如下：

```text
用户填写问卷
    ↓
独立站后端校验用户权限和数据
    ↓
后端读取私密 API 密钥
    ↓
调用 AI 模型完成提取、评分和报告生成
    ↓
校验模型返回的固定数据结构
    ↓
保存报告并返回用户可见内容
```

### 3. 为什么需要服务端

- 保护你的 API 密钥和账户费用。
- 确保只有获得邀请权限的用户可以发起诊断。
- 控制每个邀请码的诊断次数和模型成本。
- 保存问卷、评分、报告版本和人工复核记录。
- 后期可以切换 OpenAI 或其他模型供应商，而不重做前端。

### 4. 模型返回格式

模型不直接返回一篇不可控的长文章，而是按照预先定义的数据结构返回：

```json
{
  "overall_score": 76,
  "confidence": 82,
  "dimensions": {
    "positioning": {
      "score": 27,
      "status": "较清晰",
      "evidence": [],
      "gaps": []
    },
    "value": {
      "score": 25,
      "status": "基本成立",
      "evidence": [],
      "gaps": []
    },
    "trust": {
      "score": 24,
      "status": "证据较充分",
      "evidence": [],
      "gaps": []
    }
  },
  "report_sections": {},
  "needs_human_review": false
}
```

服务端需再次检查：

- 总分是否等于三个维度分数之和。
- 各维度是否超过 35、35、30 的上限。
- 每个结论是否能对应用户答案或证据。
- 待验证信息是否被误写成确定事实。
- 输出是否包含完整报告所需字段。

OpenAI 官方文档建议文本应用优先使用 Responses API；Structured Outputs 可以要求结果遵循指定 JSON Schema。API 密钥应通过服务端环境变量保存，不能部署在浏览器或移动端。

参考：

- [OpenAI Responses API 与文本生成](https://developers.openai.com/api/docs/guides/text)
- [OpenAI 最新模型指南](https://developers.openai.com/api/docs/guides/latest-model)
- [OpenAI Structured Outputs](https://developers.openai.com/api/docs/guides/structured-outputs)
- [OpenAI API Quickstart](https://developers.openai.com/api/docs/quickstart)
- [OpenAI API 密钥安全建议](https://help.openai.com/en/articles/5112595-best-practices-for-api-key-safety)

### 5. 人工复核预留

每份报告保留以下状态：

- `ai_generated`：AI 已生成，可直接交付。
- `review_requested`：用户或管理员申请人工复核。
- `under_review`：正在人工复核。
- `reviewed`：已完成人工复核。

首期默认在 `ai_generated` 后直接交付。未来可以对专家版、高价订单、低可信度报告或系统自动识别的风险报告开启人工审核。

## 十一、视觉方向与名称备选

### 1. 视觉方向

整体采用简约、克制、舒适的生活方式品牌气质，可以参考 MUJI 的信息秩序和留白方式，但不复制其标识、版式或商业视觉资产。

- 背景以暖白、米白和浅灰为主，避免纯白带来的生硬感。
- 主文字使用炭黑或深灰，辅助色使用低饱和棕、橄榄绿或砖红。
- 使用大量留白、清晰网格和自然的段落节奏。
- 减少渐变、玻璃效果、强阴影和炫技动画。
- 图表保持安静、准确，使用细线、低饱和色块和清晰数字。
- 表单像一本循序渐进的品牌工作手册，而不是传统后台系统。
- 报告页面重视阅读感，适合长文浏览和 PDF 导出。

### 2. 名称备选

| 名称 | 含义与气质 |
| --- | --- |
| 品牌三问 | 直接对应“是谁、为何选择、为何相信”，容易理解和传播 |
| 品鉴 BrandScope | 有检查与洞察含义，适合向专业工具发展 |
| 品牌知衡 | 强调认知、评估和平衡，气质克制稳重 |
| 本牌 BrandSelf | 强调看清品牌本身，适合个人与企业共同使用 |
| 品牌见 BrandView | 表达看见现状、发现问题，名称简洁 |
| 三度品牌 | 对应定位、价值、信任三个维度，具有方法论感 |
| BrandCheck 品牌检 | 功能明确，偏工具型和国际化 |
| 见本 | 看见品牌的根本，简短但需要副标题解释 |

产品名称确定为“老黄的品牌三问”，副标题为“品牌战略诊断工具”。正式上线前仍需进行名称检索和商标判断。
